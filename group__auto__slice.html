◆ auto_slice__clear()

void auto_slice__clear ( AutoSlice *  auto_slice)

This method will clear an AutoSlice structure, and the underlying memory contained in auto_slice->slice.data will be freed. However, if the elements contained by the slice field are themselves dynamically-allocated, then they must be freed prior to calling this method, as you may lose your reference when auto_slice->slice.data is freed. This method will also set the AutoSlice's allocator to NULL. The allocator field is borrowed rather than owned, so it will not be deinitialized by calling this method.

Parameters
auto_sliceA pointer to the AutoSlice structure which will be cleared.

◆ allocator

Allocator* AutoSlice::allocator

The allocator used for memory management. This is borrowed, not owned by the AutoSlice. That is, it will not be freed upon calling auto_slice__free.
